## DNS in Detail

### What I learned

* How the Domain Name System works
* How domain names are translated into IP addresses
* The DNS resolution process
* The role of DNS servers on the internet

### Key Concept

Example:

User types:

google.com

DNS converts it into an IP address such as:

142.250.x.x

This process allows browsers to communicate with servers.

Understanding DNS is important for detecting:

* Malicious domains
* Phishing attacks
* DNS-based attacks

## Pyramid of Pain

### What I learned

The **Pyramid of Pain** explains how difficult it is for attackers to change different indicators of compromise.

Levels include:

* Hash values
* IP addresses
* Domain names
* Network artifacts
* Tools
* TTPs (Tactics, Techniques and Procedures)

The higher the indicator in the pyramid, the harder it is for attackers to change it.

Security teams focus on detecting higher-level indicators because they cause more disruption to attackers.

## SOC L1 Alert Triage

### What I learned

This room explains how **SOC Level 1 analysts** investigate security alerts.

Key steps include:

1. Initial Actions
   Assign the alert and review its details.

2. Investigation
   Analyze logs from SIEM or EDR tools.

3. Final Actions
   Determine if the alert is:

* True Positive (real threat)
* False Positive (benign activity)

4. Documentation
   Write a report explaining the investigation.
